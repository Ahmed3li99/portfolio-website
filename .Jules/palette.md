## 2025-02-23 - High Contrast Focus Visible State Required for Custom Cursors
**Learning:** This app uses a custom cursor implementation (`cursor: none` on the `body`) with a trailing div element for a specialized cursor effect. When overriding the native cursor, it's critical to realize that some browser-default visual feedback for accessibility—specifically, the default focus ring for keyboard navigation (`:focus-visible`)—can be obscured or otherwise less apparent without the native cursor pointer context to draw the eye, and depending on custom theming, default focus rings often lack sufficient contrast.
**Action:** Always provide an explicit, high-contrast `:focus-visible` CSS rule (e.g., using a bright theme variable like `--cyan` with an `outline-offset`) to ensure keyboard navigators can clearly see the current focus state on interactive elements, especially when custom cursors are employed.
